# ขั้นตอนการนำเว็บไซต์ขึ้นใช้งาน (Deployment Guide)

ยินดีด้วยครับ! เว็บไซต์ของคุณสร้างไฟล์สำหรับ Production (Build) เรียบร้อยแล้ว ขั้นตอนต่อไปคือการนำไปวางบนเซิร์ฟเวอร์ที่คุณต้องการครับ

## 1. สิ่งที่ต้องอัปโหลดขึ้นเซิร์ฟเวอร์ (Files to Upload)
ให้คุณคัดลอกไฟล์และโฟลเดอร์เหล่านี้ไปไว้บนเซิร์ฟเวอร์:
- โฟลเดอร์ `dist/` (ไฟล์หน้าเว็บที่ Build เสร็จแล้ว)
- ไฟล์ `server.ts` (ไฟล์เซิร์ฟเวอร์หลัก)
- ไฟล์ `package.json` และ `package-lock.json`
- ไฟล์ `service-account.json` (สำหรับเชื่อมต่อ Google Sheets)
- ไฟล์ `.env` (ระบุตัวแปรต่างๆ เช่น CLIENT_ID)
- โฟลเดอร์ `data/` (ถ้าคุณต้องการรักษาข้อมูลห้องเรียนเดิมไว้)

## 2. การติดตั้งฝั่งเซิร์ฟเวอร์
ที่เซิร์ฟเวอร์ของคุณ (ต้องมี Node.js ติดตั้งไว้แล้ว) ให้รันคำสั่ง:
```bash
npm install --production
```

## 3. การรันเว็บไซต์ (How to Start)
ในการรันบนเซิร์ฟเวอร์จริง (Production) ให้ใช้คำสั่ง:
```bash
NODE_ENV=production npx tsx server.ts
```
*(หากต้องการให้รันค้างไว้ตลอดเวลา แนะนำให้ใช้เครื่องมืออย่าง `pm2` ครับ)*

## 4. ข้อควรระวัง (Important Notes)
1. **Google Console:** อย่าลืมไปเพิ่ม URL ของเซิร์ฟเวอร์คุณใน "Authorized redirect URIs" ในหน้า Google Cloud Console ด้วยนะครับ
2. **Data Backup:** ไฟล์ข้อมูลห้องเรียนจะถูกเก็บไว้ในโฟลเดอร์ `data/classrooms.json` อย่าลืมสำรองข้อมูลส่วนนี้ไว้เสมอครับ
3. **Port:** ปัจจุบันเซิร์ฟเวอร์รอยังคงใช้ PORT ตามที่กำหนดใน `.env` หรือค่าเริ่มต้นคือ 3000 ครับ

มีข้อสงสัยขั้นตอนไหนเพิ่มเติม หรือต้องการให้ผมช่วยเซ็ตอัพไฟล์ไหนเป็นพิเศษ บอกได้เลยนะครับ!
